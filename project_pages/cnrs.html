Production d'une API de clustering & représentation 3D pour le CNRS

Dans le cadre d'un travail de recherche en archéologie pour le CNRS, F80 a développé une API de clustering et d'analyse de graphe dans le but d'opérer certains rapprochement d'échantillons archéologiques par composition chimique similaire.

L'api, développée en Python, repose sur la librairie SCIKIT-LEARN pour le clustering et NetworkX pour l'analyse de graphe. Elle utilise les derniers algorithmes de clustering (notamment l'optimisation de DBSCAN en multi-densité : HDBSCAN) pour regrouper des échantillions archélogiques par composition chimique similaire.

La projection 3D utlise des techniques de projection reposant sur l'analyse en composantes principales pour représenter des vecteurs à "n" dimensions. Pour garantir un fonctionnement sur tout type de terminaux, la représentation 3D repose sur la bibliotheque BabylonJS, habituellement utilisé pour le jeu vidéo et compatible avec tous les navigateurs récents du marché.

On assure ainsi une parfaite représentation tout en permettant une certaine interactivité sur le graphique (changement d'angle, mise en transparence, enregistrement vidéo ...)

Du point de vue infrastucture, l'api peut être déployée rapidement sur n'importe quel serveur x86 via Docker. Ainsi le calcul peut être déporté vers des machines dédiées pendant que la représentation et la manipulation des données reste accessible sur tout type de terminaux (y compris les smartphones).

Exemple d'utilisation :
https://server.f80.fr:5000/graph/lesmis.gml/fr?algo_comm=NOTREATMENT&dir=public¬ext=True&nometrics=True&add_property=False&autorotate=True&limit=5000&pca=1&processors=2

Les réprésentations ci-dessous sont directement manipulables depuis la souris et le clavier (voir les FAQ pour plus d'informations)
