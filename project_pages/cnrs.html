Production d'une API de clustering & représentation 3D pour le CNRS

L'api, développée en Python, repose sur la librairie SCIKIT-LEARN pour le clustering et NetworkX pour l'analyse de graphe. Elle utilise les derniers algorithmes de clustering pour regrouper des échantillions archélogique par composition chimique proche.
Certains algorithme tels qu'HDBSCAN (une optimisation de DBSCAN en multi-densité) sont en phase béta. Quelque soit le nombre de composantes, les points sont projetés dans un espace en 3 dimension par analyse en composante principale, pour être représenté en 3D dans n'importe quel navigateur WEB par la mise en oeuvre de la bibliotheque BabylonJS, habituellement utilisé pour le jeu vidéo. On assure ainsi une parfaite représentation tout en permettant une certaine interactivité sur le graphique (changement d'angle, mise en transparence, enregistrement vidéo ...)

Du point de vue infrastucture, l'api peut être déployé rapidement sur n'importe quel serveur x86 via Docker
Exemple d'utilisation :
https://ss.shifumix.com:5000/graph/lesmiserables.gml/fr?algo_comm=NOTREATMENT&dir=public¬ext=True&nometrics=True&add_property=False&autorotate=True&limit=5000&pca=1&processors=2
